# Sample Output Shape

This is a shortened example that shows the expected tone and section layout.

```md
# APP-2048 Demo 稿：前台预约信息在列表中提前可见

## 一句话总结

这次更新把预约关键信息前置到用户最常看的列表入口，让 frontdesk 不需要反复点进详情就能做更快判断。

## 1. User Story

**Jira Story**

As a frontdesk user, I want to see appointment payment status directly in the appointment list so that I can confirm customer status faster.

**中文表述**

作为前台同学，我希望在 appointment list 里直接看到 payment status，这样我在接待客户时可以更快判断下一步动作，不需要来回进入详情页确认。

## 2. 这次需求解决了什么问题

以前前台需要先进入 appointment drawer 才能确认支付状态，操作路径长，而且在高峰时段容易漏看。

## 3. Solution / Outcome

现在 payment status 在列表入口就能看到，frontdesk 可以更快完成判断，减少反复进入详情页的动作。
```
